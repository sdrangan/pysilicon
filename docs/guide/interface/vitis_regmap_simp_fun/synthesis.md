# Synthesis and timing

Run the full flow:

```bash
cd examples/interface/vitis_regmap_simp_fun
python simp_fun_build.py --through generate_timing_diagram
```

## Stages after Python simulation

- `gen_kernel` and `gen_tb` generate the HLS kernel and C++ testbench.
- `csim` runs the Vitis C-simulation testbench.
- `validate_csim` compares `y` and `status` against the Python outputs.
- `csynth` runs C-synthesis and RTL co-simulation.
- `extract_cosim_timing` parses the Vitis co-sim report.
- `validate_timing` checks Python-vs-cosim cycle agreement.
- `generate_timing_diagram` writes the final diagram artifacts.

## Timing artifacts

- `results/cosim_timing.json`
- `results/timing_verdict.json`
- `results/timing_diagram.svg`
- `results/timing_diagram.json`

The SVG is generated by a normal DAG step, not by a separate ad-hoc script. The JSON companion records the event annotations and the cycle counts used to render the diagram.

## Reading the verdict

`timing_verdict.json` reports:

- Python cycle count
- co-sim cycle count
- absolute delta
- configured tolerance
- pass/fail bit

That makes the example useful both as a teaching flow and as a small regression target for future timing-model work.
